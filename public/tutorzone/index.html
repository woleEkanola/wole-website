Live Classroom (Built-In)
LiveKit → embedded video → bidirectional screen sharing → whiteboard → emoji chat → virtual background → silent observer

What It Does

Embedded live video classroom powered by LiveKit. Supports switching between self-hosted LiveKit (Apache 2.0) and LiveKit Cloud via configuration. Supports 1-on-1 and group sessions with bidirectional screen sharing (both tutor and students), interactive whiteboard for writing letters, mathematical expressions, equations and diagrams, emoji support in chat, fixed Tutorszone virtual background for tutors, mute/hand-raise, and silent observer mode for the Learning Experience Specialist (QA). Attendance tracked automatically via LiveKit's participant events API.

LiveKit: Self-Hosted vs Cloud

Self-Hosted LiveKit

  • You deploy the SFU server
  • You manage coturn (TURN/STUN)
  • Full control over infrastructure
  • Cost: $12–18/month (server)
  • Best for: full control, no per-minute fees
⇄ Switch via config
Environment variable

LiveKit Cloud

  • LiveKit manages the SFU
  • No server to maintain
  • Auto-scaling, global CDN
  • Cost: $0.002/min (free tier: 50 hrs/mo)
  • Best for: fast launch, no ops burden

Frontend code is identical in both cases. The LiveKit React SDK connects to whichever server URL and API key you configure. The only difference is infrastructure management.

Masked Calling with Recording
Teacher initiates call → number masked → audio recorded → deleted after 7 days

What It Does

Teachers can initiate a one-click phone call to a parent directly from their dashboard without seeing the parent's phone number. The parent's caller ID shows a Tutorszone number. All calls are automatically recorded (audio only) and stored securely. Recordings are accessible only to LES and Super Admin for quality assurance. All recordings are deleted after 7 days.

Use Case: Late to Class

When a parent or teacher is late to a scheduled class, the party who is present can click a button to trigger a voice call to the other party. This call is recorded for QA review. This is the primary use case for masked calling in MVP.

Feature Breakdown

One-Click Call Initiation Teacher clicks "Call Parent" button on student profile card. System initiates call via Twilio Proxy.
Number Masking Teacher never sees parent's phone number. Parent sees Tutorszone number (not teacher's personal number).
Auto Recording All calls automatically recorded server-side via Twilio. Audio only (no video).
Secure Storage Recordings stored locally on app server. Accessible only to LES and Super Admin.
7-Day Retention All recordings automatically deleted after 7 days. Cron job handles cleanup.
Call Log Teacher sees call history: date, time, duration only (no phone number). LES/Super Admin see full log with recording link.
QA Review LES can playback recordings, rate call quality (professionalism, clarity, outcome), and flag for coaching.
Opt-Out Support If parent has opted out of calls, the "Call Parent" button is disabled on teacher dashboard.

Tutor Screen Monitoring
Auto-screenshot every 10 min → timeline view for LES → Admin/LES view all teaching resources → deleted after 7 days

What It Does

During live classes, the platform automatically captures screenshots of the tutor's shared screen at configurable intervals (default: every 10 minutes). Screenshots are stored securely and accessible only to LES and Super Admin for quality assurance review. LES can review screenshots in a timeline view per session. All screenshots are deleted after 7 days.

How It Works

Screenshots capture the tutor's shared screen (what students see), not their full desktop. This is done by subscribing to the tutor's screen share track via LiveKit's API and extracting frames. Lightweight, no extra infrastructure needed.

Feature Breakdown

Auto-Capture Backend worker captures screenshot every 10 minutes during active sessions. Configurable interval by Super Admin.
Shared Screen Only Captures the tutor's shared screen (teaching content), not their full desktop. Less invasive, more relevant for QA.
Secure Storage Screenshots stored locally on app server. Access limited to LES and Super Admin only.
Timeline View LES can review screenshots in a timeline view per session, per tutor. Thumbnails with timestamps.
7-Day Retention All screenshots automatically deleted after 7 days. Cron job handles cleanup.
QA Review LES uses screenshots to verify: lighting, background, uniform, professionalism, teaching content.
Configurable Frequency Super Admin can configure screenshot interval (e.g., every 5, 10, 15 minutes).
Transparency Notice Tutors are notified that screen monitoring is active during sessions (shown in classroom UI).
Teaching Resources Visibility Admin/LES can view all teaching resources (videos, links, PDFs, screenshots) uploaded by tutors across all sessions. Centralized resource library for quality oversight and compliance.

Key Decisions

These decisions must be made before engineering starts. Each one blocks or shapes a significant portion of the architecture.

1. LiveKit: Self-Hosted vs Cloud

BLOCKS MVP DECIDED

The platform must support switching between self-hosted LiveKit and LiveKit Cloud via configuration. Both use the same LiveKit SDK on the frontend — only the server URL and API key change.

FactorSelf-HostedLiveKit Cloud
InfrastructureYou deploy and manage the SFU serverLiveKit manages everything
Cost$12–18/month (server) + bandwidth$0.002/min (free tier: 50 hrs/mo)
Ops BurdenMedium (Docker, coturn, monitoring)None (fully managed)
ControlFull control over infrastructureLimited to LiveKit's platform
ScalingYou manage scaling (vertical/horizontal)Auto-scaling, global CDN
Best ForLong-term cost savings, full controlFast launch, no ops burden
Decision: Support both via configuration. Start with LiveKit Cloud for MVP (fastest to launch, no ops burden). Migrate to self-hosted in V2 once the platform is stable and you want to reduce per-minute costs at scale. The frontend code is identical — only the config changes.

2. Screen Monitoring Approach

BLOCKS MVP DECIDED

Screenshots every 10 minutes during live sessions. Deleted after 7 days. Captures the tutor's shared screen (what students see), not their full desktop.

Shared screen ✅ Capture the tutor's shared screen via LiveKit's track subscription. Lightweight. Only captures what students see.
Full desktop Headless browser joins session, captures full video frame. Heavier. Captures tutor's video feed, not their actual desktop.
Browser extension Tutor installs extension that captures their screen. Most invasive. Requires tutor cooperation.
Decision: Shared screen capture via LiveKit track subscription. Captures what students see (the teaching content). Lightweight, no extra infrastructure. If full desktop monitoring is needed later, add Puppeteer-based capture in V2.

3. WhatsApp Notification Channel

BLOCKS MVP PENDING

WhatsApp is required for class reminders (15 min before), schedule changes, and notifications. How do we integrate?

Via Twilio ✅ Use Twilio's WhatsApp API. Easier approval, better docs, template management. Higher cost per message (~$0.005/msg).
Direct Meta API Apply directly through Meta for WhatsApp Business API. Lower cost but strict approval process (2–4 weeks). Requires business verification.
Recommendation: Via Twilio for MVP. Faster to set up, better documentation, and WhatsApp is available in all Tutorszone operating countries (USA, Canada, UK, UAE, SA, Nigeria) via Twilio. Switch to direct Meta API in V2 for cost savings at scale.

4. Masked Calling Provider

BLOCKS MVP DECIDED

Teachers initiate calls to parents without seeing their phone number. Calls are recorded (audio only) and deleted after 7 days.

Twilio Proxy ✅ Purpose-built for marketplace-style masked calling. ~$0.01/session + per-minute charges. Good Nigeria coverage. Recording built-in.
Vonage Number Masking Similar capability. Slightly cheaper per-minute but less documentation and smaller community.
Decision: Twilio Proxy. Best docs, most mature proxy API, good coverage in all Tutorszone countries including Nigeria. Built-in recording simplifies the QA workflow.

5. Student Age Range

BLOCKS MVP PENDING

What age range are the students? This determines the complexity of the student dashboard UI.

6–10 years Needs very visual, simple UI. Large buttons, minimal text. Possibly parental co-use required.
11–18 years ✅ Can handle text-based UI. Simple but functional. Independent use.
Mixed (6–18) Need two UI modes: simple (young) and standard (teens). More design and development work.
Recommendation: 11–18 years for MVP. Students in this range can use a standard dashboard independently. If younger students (6–10) are a significant segment, plan for a simplified "junior" UI in V2.

Infrastructure

Designed to switch between self-hosted LiveKit and LiveKit Cloud. No video recording = dramatically reduced costs. Screenshots and audio calls are stored locally and deleted weekly.

LiveKit Switching Architecture

The platform uses a configuration-based approach to switch between self-hosted LiveKit and LiveKit Cloud. The frontend code is identical in both cases — only the server URL and API key change.

Configuration

Switching is controlled by environment variables. Set LIVEKIT_MODE to either self_hosted or cloud, then provide the corresponding URL, API key, and API secret. The frontend SDK connects to whichever endpoint is configured — no code changes required.

VariableSelf-HostedCloud
LIVEKIT_MODEself_hostedcloud
LIVEKIT_URLwss://livekit.yourdomain.comwss://your-project.livekit.cloud
LIVEKIT_API_KEYYour self-hosted API keyYour LiveKit Cloud API key
LIVEKIT_API_SECRETYour self-hosted API secretYour LiveKit Cloud API secret

Break-even point: At ~1,500 hours/month (~125 students), self-hosted becomes cheaper than LiveKit Cloud. Below that, cloud is more cost-effective (no server to manage). Above that, self-hosted saves money.

Recommendation: Start with LiveKit Cloud for MVP. Migrate to self-hosted when you hit 100+ students and want to reduce costs.

Free Tier Option (Oracle Cloud)

Oracle Cloud offers always-free ARM instances (4 vCPU, 24GB RAM total across instances). You can run the entire stack on free tier:

  • ARM Instance 1: App server — Next.js + PostgreSQL + Redis (4 vCPU, 12GB RAM)
  • ARM Instance 2: LiveKit SFU + coturn (4 vCPU, 12GB RAM) — if self-hosting
  • Object Storage: 10GB free (for backups)

Total cost: $0/month — ideal for MVP validation. Limitations: no guaranteed availability, instances can be reclaimed under heavy load.

Alternative: Use LiveKit Cloud (free tier: 50 hrs/month) + Oracle Cloud free tier for app server = $0/month with better reliability.
